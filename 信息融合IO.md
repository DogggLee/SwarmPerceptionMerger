# 模块输入
## 背景信息（指控提供）
 - 传感器配置集合
    所有用到的传感器模态集合，以及其在json中的enum标识（如RADAR=0, RGB=1, IF=2, ELEC=3）
    每个传感器的感知范围（水平、垂直视场角，有效感知距离等）、图像分辨率等信息

 - 感知类别集合
    每个模态传感器的所有感知目标类别集合，以及其在json中的enum标识（如 PERSON=0, TANK=1）

 - 类别关联表
    每个传感器模态-目标类型，其对应在每个传感器模态下，可能存在关联的类别集合：（同模态默认只关联同类别目标，也可以设置相近类目）
    dict[sensor_type][class_id][sensor_type] = [class1, class2, class3]

 - 其他关联规则
    除位置、可视范围、类别关联以外，其他的特殊模态相关关联规则   如雷达特有属性、红外特有属性对于判别目标关联的具体策略规则

## 基础配置文件
 配置了所有信息融合所需相关参数，包括但不限于
 - 距离阈值（平面距离阈值与高度阈值）
 - 速度夹角阈值
 - 可视角度阈值
 - 感知距离阈值
 - 轨迹关联程度阈值

## 即时目标感知结果
 集群单个无人机的某次目标感知结果（每次只有单个模态的感知结果上报）

 - 无人机ID
 - 传感器类型
 - 无人机全局位置（东北天坐标）
 - 无人机全局姿态（RPY坐标）
 - 感知时间戳
 - 感知结果列表：
    - 目标类别
    - 目标全局位置
    - 目标全局速度
    - 目标Track ID
    - 目标图像

## 全局态势目标信息（指控维护）
 当前指控系统中维护的所有任务目标信息记录，每条记录应当按照特定时间间隔进行更新（更新位置、速度方向和历史轨迹），其最终更新的时间戳与即时目标感知结果的时间差异<= 1s (太长难以满足匀速运动假设)。所有任务目标信息不应当有过旧的失效记录，以避免导致错误关联。

 每条任务目标信息记录包括：
 - 目标全局ID
 - 目标全局位置（东北天坐标）
 - 目标全局姿态（RPY坐标）
 - 记录最终更新时间戳
 - 目标历史轨迹  (包括历史观测位置以及预测的位置组成的轨迹)
 - 目标类别集合（其在每个模态下的class_id）
 - 历史观测记录（所有与其关联的历时即时目标感知结果信息）
   - 无人机ID
   - 传感器类型
   - 传感器全局位置（东北天坐标）
   - 传感器全局姿态（RPY坐标）
   - 感知时间戳
   - 目标类别
   - 目标全局位置
   - 目标全局速度
   - 目标Track ID

# 模块输出
分为 成功匹配目标列表，与新增目标列表 两组信息
 - 成功匹配目标列表（与全局态势目标信息成功匹配关联）
   - 目标ID
   - 目标融合后位置
   - 目标融合后速度
   - 目标新增观测信息记录
     - 无人机ID
     - 传感器类型
     - 传感器全局位置（东北天坐标）
     - 传感器全局姿态（RPY坐标）
     - 感知时间戳
     - 目标类别
     - 目标全局位置
     - 目标全局速度
     - 目标Track ID

 - 新增目标列表
   - 目标ID
   - 目标位置
   - 目标速度
   - 目标观测信息记录
     - 无人机ID
     - 传感器类型
     - 传感器全局位置（东北天坐标）
     - 传感器全局姿态（RPY坐标）
     - 感知时间戳
     - 目标类别
     - 目标全局位置
     - 目标全局速度
     - 目标Track ID

# 待定问题
1. 不同模态下的目标类型不一致，在全局态势信息中，每个目标的类别以哪个为准？
2. 上级友邻提供的输入信息，与即时目标感知结果存在什么差异？
3. 全局态势信息中新增的目标记录只有一条有效观测记录，指控端在实际显示目标时，应该只使用历史有效观测记录数量>某阈值的目标观测，用于处理虚警情况。当距离当前时间大于某阈值的观测记录视作无效观测记录，应该被剔除，当目标记录没有任何有效观测记录后，应该直接从全局态势信息中被删掉。（动目标和静目标的阈值设定会有所不同）
（在信息融合时，有效观测数量>0的所有目标记录都要提供给信息融合模块）
4. 全局态势信息在定期更新时，可能与信息融合的计算过程冲突，要考虑避免采用异步更新，避免融合结果与预测结果冲突。
5. 目标感知中的目标位置、速度方向需要足够准确  （同一传感器短时间内的连续目标位置对应轨迹运动方向，与实际轨迹不存在差异；同一目标的运动方向在不同传感器感知结果里，误差应当小于5度）
