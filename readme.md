# Perception Merger
在多无人机集群应用中，每个无人机可能会配备不同模态的感知结果，并回传自身的即时目标感知结果到指控中心，指控中心需要根据回传结果进行信息融合（时空对齐、身份关联与预测融合）

# 核心类
## GlobalInfo
在指控中心维护当前所有已发现的任务目标记录，其由多条任务目标记录组成，每条记录包括：
 - 全局ID
 - 目标class_id
 - 目标全局位置
 - 目标全局速度
 - 目标历史轨迹
 - 目标历史观测记录

另外该类还具备几项基础功能：
 - 检索：根据目标的全局ID、位置、类别检索所有符合要求的目标记录
 - 编辑：根据目标全局ID编辑对应目标记录内容
 - 新增：新增一条目标记录，并为其分配全局ID
 - 删除：删除某个目标记录
 - 预测：根据目标历史观测记录、历史轨迹、当前位置、全局速度等信息，预测该目标在指定时间戳下的位置与速度；同时根据某些规则，标记历史观测记录过少 / 过旧 的目标记录，将其标注为不可信或失效

在得到PerceptionMerger的融合结果后，对任务目标记录进行更新

## PerceptionMerger
作为信息融合的核心，从GlobalInfo中读取当前指控中心的所有有效目标记录信息，并与当前得到的即时目标感知结果进行融合，确认即时感知结果与当前目标记录的关联关系，并返回其匹配结果和融合后的信息。

其还需要事先建立一个类别关联表，说明不同模态传感器的感知类别之间的潜在关联关系，因为不同模态的传感器在感知类别上很可能不一致，但是部分类别之间有一定的重叠关系，如可见光相机感知的"车辆"类别，与红外相机中的"异常高温/低温"等类别有较高关联。

即时目标感知结果数据组成为：
 - 无人机ID（全局唯一，提前分配）
 - 传感器类型
 - 无人机位置
 - 传感器朝向
 - 传感器参数（视场角等参数）
 - 目标感知结果列表：
    - 目标class_id
    - 目标全局位置
    - 目标全局速度
    - 目标track ID (仅在同无人机ID、同传感器类型下唯一，且track ID可能受到遮挡、误检等因素干扰影响)
    - 目标2D BBox
    - 时间戳

整个融合过程包括以下几个步骤
 - 对齐: 根据即时目标感知结果的时间戳，对读取的全局目标记录信息进行时空对齐（根据其当前位置、速度和时间差，基于匀速运动假设进行对齐）
 - 单机感知轨迹查询：根据无人机ID、传感器类型和Track ID，查找得到该track ID的历史运动轨迹（由PerceptionMerger内部维护）
 - 关联：求解即时目标感知结果中的每个目标感知结果与全局目标记录的关联关系
    - 计算匹配代价：计算每个目标感知结果与全局目标记录的关联程度，其需要基于以下几个信息
      - 类型相关度：根据类别关联表判断类别是否存在关联关系，不存在关联的匹配代价无穷大
      - 3D距离相关度：位置越接近，关联程度越高，大于某个阈值的代价无穷大
      - 可视程度：计算当前全局目标位置，是否可能出现在当前时间戳下的传感器视场角范围内，或与其2D BBox的像素距离（要求无人机位姿足够准确，否则无意义）
      - 轨迹相关度：根据track ID轨迹与全局目标记录对应的历史轨迹计算其关联程度（轨迹重合度，整体运动方向等）
    - 求解匹配关系：根据匹配代价矩阵求解即时目标感知结果与全局目标记录之间的匹配关系。成功匹配的视作与全局目标记录关联，否则视作新发现目标
 - 更新：根据关联结果，将即时目标感知结果作为新的观测，对目标记录信息进行更新（利用KF更新其位置、速度与轨迹），增加其对应的感知记录
      
JsonIO: 实现json数据的http请求写入与读取。

可以将即时目标感知结果（Json格式）打包进行发送、从指定url中读取；  可以将从GlobalInfo中读取的全局目标记录集合打包发送、读取。   可以将PerceptionMerger的返回结果进行发送、读取

# 特殊约束
在实际任务中，信息融合需要具备以下几个情况的处理能力：
1. 对于某些类别，会事先知道其总数，在信息融合过程中可以考虑利用该信息，并对矛盾的情况进行及时告警
2. GlobalInfo需要定期预测目标在新时间戳下的状态，但是在没有新的观测进入的情况下，其预测结果势必会有巨大噪声，因此在融合过程中，需要适当考虑该问题，在最终进行更新时考虑新旧信息的权重（KF是否可以自动处理该情况？）
3. 当前主要工作模式为任意一路感知结果传入后，立刻与全局目标记录进行融合（即所有感知结果串行处理，因此对时效性要求较高）；   未来可能会变成积攒1s以内的所有感知结果进行批量处理，此时的融合策略可能会有所不同，需要为perceptionMerger设计额外的响应接口
