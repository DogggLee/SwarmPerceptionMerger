# 融合策略
整个融合过程包括以下几个步骤
 - 对齐: 根据即时目标感知结果的时间戳，对读取的全局目标记录信息进行时空对齐（根据其当前位置、速度和时间差，基于匀速运动假设进行对齐）
 - 单机感知轨迹查询：根据无人机ID、传感器类型和Track ID，查找得到该track ID的历史运动轨迹（由PerceptionMerger内部维护）
 - 关联：求解即时目标感知结果中的每个目标感知结果与全局目标记录的关联关系
    - 计算匹配代价：计算每个目标感知结果与全局目标记录的关联程度，其需要基于以下几个信息
      - 类型相关度：根据类别关联表判断类别是否存在关联关系，不存在关联的匹配代价无穷大
      - 3D距离相关度：位置越接近，关联程度越高，大于某个阈值的代价无穷大
      - 可视程度：计算当前全局目标位置，是否可能出现在当前时间戳下的传感器视场角范围内，或与其2D BBox的像素距离（要求无人机位姿足够准确，否则无意义）
      - 轨迹相关度：根据track ID轨迹与全局目标记录对应的历史轨迹计算其关联程度（轨迹重合度，整体运动方向等）
    - 求解匹配关系：根据匹配代价矩阵求解即时目标感知结果与全局目标记录之间的匹配关系。成功匹配的视作与全局目标记录关联，否则视作新发现目标
 - 更新：根据关联结果，将即时目标感知结果作为新的观测，对目标记录信息进行更新（利用KF更新其位置、速度与轨迹），增加其对应的感知记录
