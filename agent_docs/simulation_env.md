# 集群仿真环境
环境成员：
 - 侦察无人机：每个侦察无人机配备有对应的感知传感器，可以感知其感知范围内的目标
    - 传感器模态：
      - 可见光：感知范围为无人机机头前方一定距离的矩形区域。 可以感知目标类别、位置、速度和Track ID。  
      - 红外：感知范围为无人机机头前方一定距离的矩形区域。 可以感知目标类别、位置、速度和Track ID。  
      - 雷达：感知范围为无人机为中心的圆形区域。 可以感知目标类别、位置、速度 
      - 电子侦察：感知范围为无人机为中心的圆形区域。 可以感知目标类别和位置方向（无准确位置，只能感知目标在以侦察无人机为起点，指向任务目标的射线方向上）

- 任务目标
    - 类别：其在不同模态传感器下均有对应的类别
    - 行动模式：
      - 随机：按一定间隔随机游走，运动速度方向随机
      - 巡逻：按固定路线往返巡逻，运动速度在指定区间内平滑变化
      - 固定：某些特定类别静止目标（如建筑）只能原地固定，运动目标则也有概率采取静止动作。

环境元素
 - 气候：随机区域有天气分布：晴天、阴天、雨天、雪天
    - 雨雪天气中可见光失效
 - 整体光照：白天，黑夜
    - 白天红外失效
    - 夜间可见光失效
    
# 仿真过程
## 环境初始化
通过参数设置，创建一个指定大小的全局环境地图（地图只以2D显示，x轴为东，y轴为北，但是目标的位置和速度则是3D坐标），并放置N个任务目标，其对应的行动模式随机生成配置。

放置M个侦察无人机，每个侦察无人机只配备一种模态传感器。  按照提前规划的轨迹分配每个侦察无人机的航线，实现对任务区域的覆盖式搜索侦察。

即时感知结果缓存池初始化为空
全局态势信息初始化为空

## 执行过程
任务目标与侦察无人机各自执行自身的路线。

当任务目标出现在侦察无人机的侦察范围内（仅考虑x-y坐标是否覆盖，但是感知结果为3D信息）后，将会根据任务目标属性生成对应的感知结果进行上报，存储到即时感知结果缓存池中。

## 融合过程
信息融合模块会持续从即时感知结果缓存池中读取最先加入到缓存的即时感知结果，以及全局态势信息进行信息融合，并根据融合结果对全局态势信息进行更新。

## 预测过程
全局态势信息在任意一次更新后，每隔0.5秒更新其内部目标记录的位置速度轨迹等信息，并剔除每条目标记录的失效历史观测记录，以及完全失去所有有效观测记录的目标记录。
